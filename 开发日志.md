# 三资辅助软件开发日志

## 项目概况

- 软件名称：三资辅助软件
- 当前版本：1.0
- 开发平台：Windows
- 主要技术：Python 3.12、PySide6、Qt WebEngine、Pillow、Shapely
- 打包方式：Nuitka Standalone + Inno Setup
- 首个正式版本发布日期：2026年6月19日

这个项目源于一套已经丢失源码的旧软件。开发目标不是简单照搬旧界面，而是重新梳理实际工作流程，做成一套普通用户也能看懂、可以在不同电脑上独立运行的“三资”图斑和照片辅助工具。

## 2026年6月19日：版本 1.0

### 一、重新规划整体结构

最初功能较为分散，包括照片水印、批量命名、图斑整理、KML处理和地图工具。经过重新梳理，最终将软件划分为六个工作页面：

1. 下载平台图斑
2. 制作无人机航线
3. 给照片加水印
4. 按图斑整理照片
5. 上传照片到平台
6. 查看照片地图

其中照片选择、水印设置、批量编号和照片输出被合并到同一个页面；按图斑整理照片作为独立流程保留。

界面文字尽量避免使用开发术语，改用“保留原照片”“取走原照片”“先看看整理结果”“先检查哪些照片能上传”等更容易理解的说法。

### 二、照片水印与批量命名

完成以下功能：

- 选择照片目录并支持扫描子文件夹。
- 读取照片 EXIF 中的拍摄时间、经纬度和图片尺寸。
- 在照片左下角添加标题、经度、纬度、时间和自定义文字。
- 支持调整字号、文字颜色、描边颜色、透明度和边距。
- 照片根据拍摄时间排序，从 `A1` 开始连续编号。
- 处理结果保存到用户指定的新目录，不覆盖原照片。
- 同名文件自动增加 `_1`、`_2` 等后缀。
- 支持原图和水印效果预览。

为解决照片数量较多时加载和切换卡顿的问题，增加了：

- 每页最多显示500张照片。
- 照片选择状态独立保存，翻页不会丢失。
- 预览草图和最近预览缓存。
- 设置变化延迟刷新，避免连续重复生成预览。
- 快速处理和兼容处理两种模式。
- 快速模式最多同时生成两张照片。

### 三、KML图斑解析

为了避免打包环境缺少 Fiona/GDAL KML驱动，放弃直接调用 `geopandas.read_file()`，改为使用 `xml.etree.ElementTree` 手动解析KML。

解析内容包括：

- `Placemark`
- 图斑名称
- `landcode`
- `Polygon`
- `MultiPolygon`
- 外边界坐标
- KML来源文件

无效多边形会尝试修复，多个面会合并为一个图斑记录。

### 四、按图斑整理照片

完成照片GPS与KML图斑的自动匹配流程：

1. 读取照片EXIF中的GPS。
2. 使用经度、纬度建立照片位置点。
3. 将照片点和图斑转换为米制Web Mercator坐标。
4. 判断照片是否位于图斑内部或边界上。
5. 如果没有直接命中，根据用户设置的允许距离寻找最近图斑。
6. 重叠图斑优先选择面积较小、范围更具体的图斑。
7. 将匹配成功的照片复制或移动到对应图斑文件夹。

性能方面，最初需要让每张照片遍历所有图斑，大量照片会明显卡顿。后来改用Shapely的 `STRtree` 空间索引，大幅减少无关图斑计算。

为兼容不同电脑和存储设备，增加两种模式：

- 快速整理：最多同时处理两张照片，适合SSD和性能较好的电脑。
- 兼容整理：单线程处理，适合机械硬盘、旧电脑和不稳定U盘。

整理完成后生成：

```text
图斑编号文件夹/
无照片图斑.kml
未匹配照片.kml
图斑外距离匹配照片清单.txt
图斑照片分类工作日志.txt
整理结果.csv
```

没有照片的图斑不会建立空文件夹，未匹配照片也不会额外复制，只通过KML和结果表记录。

### 五、图斑整理问题修复

开发过程中处理了以下问题：

- 点击预分析时界面卡死。
- 整理完成后软件闪退。
- 后台线程调用PROJ DLL时出现Windows原生崩溃。
- 照片复制失败后仍被统计为匹配成功。
- 输出目录重复使用导致旧照片和新照片混合。
- 无照片图斑被错误建立为空文件夹。
- 未匹配照片重复复制导致处理速度下降。
- 大量照片复制速度过慢。
- 左侧照片统计与上方结果统计不一致。

为避免PROJ DLL在后台线程中触发崩溃，坐标转换改为项目内部的纯Python Web Mercator实现，不再依赖运行时的 `pyproj`。

### 六、平台图斑下载

将原浏览器插件中“下载当前已显示图斑”的能力整合进桌面软件。

完成以下功能：

- 在软件内登录三资平台。
- 打开数据采集地图。
- 读取地图上当前已经显示的图斑。
- 将当前图斑导出为KML。
- 支持重新加载平台。
- 支持切换账号和退出登录。

修复了“提示无法读取图斑，但文件仍能正常下载”的问题。原因是PySide6接收JavaScript对象返回值时可能得到空字符串，后来统一使用JSON字符串传递结果。

### 七、平台登录状态共享

下载页面和上传页面最初各自要求登录，用户需要重复输入账号。

后来改为使用同一个持久化 `QWebEngineProfile`，共享：

- Cookie
- Local Storage
- Session Storage
- 登录Token
- 请求头中的Authorization信息

同时增加：

- 切换账号
- 退出登录
- 清理Cookie和缓存
- 自动识别当前登录地区

账号和密码只在平台网页中输入，软件不保存账号密码。

### 八、照片上传前安全校验

早期上传逻辑只检查照片文件夹名称是否包含长数字，存在把其他村庄照片错误识别为可上传的风险。

后来增加强制校验：

1. 用户必须选择本次照片对应的KML。
2. 检查当前登录地区编号。
3. 读取KML中的全部图斑编号。
4. 检查照片文件夹名称中的图斑编号。
5. 文件夹编号必须属于当前地区。
6. 文件夹编号必须存在于所选KML。
7. 任意一项不一致时阻止整批上传。

上传流程调整为：

1. 登录平台。
2. 选择KML。
3. 选择整理后的照片目录。
4. 先检查哪些照片能上传。
5. 查看右侧明细。
6. 确认后才真正上传。

检查阶段不会向平台上传照片。

### 十一、正式打包

版本1.0采用：

```text
Nuitka Standalone + Inno Setup
```

选择这种方式的原因：

- Python代码会经过C/C++转换和本机编译，比直接打包字节码更难逆向。
- Qt WebEngine使用目录模式运行更稳定。
- 用户电脑不需要安装Python或任何依赖。
- 安装程序支持全中文安装和卸载。
- 可以建立桌面和开始菜单快捷方式。

打包过程中处理了以下兼容问题：

- 中文项目路径导致Nuitka按GBK读取文件失败。
- Nuitka未完整收集新版PySide6拆分后的Qt主DLL。
- `shiboken6.abi3.dll`缺失导致程序启动失败。
- Inno Setup默认不附带简体中文语言文件。
- Qt WebEngine运行资源和子进程收集。
- HTML、QSS、JavaScript和ICO资源路径。

最终生成：

```text
三资辅助软件_1.0_安装程序.exe
三资辅助软件_1.0_免安装版.zip
发布文件校验值_SHA256.txt
```

正式产物经过以下验证：

- 编译版持续运行测试。
- Qt WebEngine子进程启动测试。
- 标准安装测试。
- 安装后启动测试。
- 静默卸载测试。
- 卸载目录清理测试。
- 免安装ZIP解压运行测试。

## 版本1.0已知限制

- 安装程序尚未购买代码签名证书，Windows可能显示“未知发布者”。
- 平台网页结构、接口地址或登录方式变化后，下载和上传功能可能需要适配。
- 在线地图功能需要网络。
- 没有GPS信息的照片无法自动判断所属图斑。
- 图斑外允许距离设置过大时，照片可能归入相邻图斑。
- 本机程序可以提高逆向难度，但无法保证完全不能被逆向。

## 2026年6月19日：照片读取增强

版本1.0发布后继续完善照片扫描能力：

- 增加照片信息持久化缓存。
- 缓存照片路径、文件大小、修改时间、GPS、拍摄时间和尺寸。
- 再次扫描时只对新增或已经变化的照片重新读取。
- 缓存采用临时文件写入后原子替换，避免软件异常退出破坏缓存。
- 缓存损坏或版本不兼容时自动忽略并重建。
- EXIF缺少GPS或拍摄时间时，自动读取JPEG内嵌XMP。
- 支持DJI常见的小数经纬度格式。
- 支持 `35,7.4N` 一类度分坐标格式。
- XMP定位异常或超出经纬度范围时不予采用。

真实照片目录测试：

- 照片数量：3745张。
- 有GPS照片：3743张。
- 首次读取约4.5秒。
- 缓存读取约2.7秒。

缓存位于 `%LOCALAPPDATA%\三资辅助软件\照片信息缓存.json`，只保存照片元数据，不保存照片内容，也不会上传网络。
