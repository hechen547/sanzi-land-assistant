# 三资辅助软件

面向农村集体“三资”图斑巡查、无人机航线规划和现场照片整理工作的 Windows 桌面工具。

软件把常用流程集中到一个界面中：下载平台图斑、规划无人机航线、给照片添加定位水印、按 KML 图斑自动整理照片、检查并上传照片，以及在地图中查看照片位置。

> 照片扫描、水印生成、批量命名、GPS 匹配和文件整理均在本机完成。
>
> 平台登录、图斑下载、照片上传以及在线地图底图需要联网。

当前正式版本：`1.0`

## 主要功能

### 1. 下载平台图斑

- 在软件内打开三资平台登录页面。
- 复用同一软件中的平台登录状态。
- 下载地图上当前已经显示的图斑并保存为 KML。
- 支持重新加载平台、切换账号和退出登录。

该功能下载的是平台地图中当前已经加载并显示的图斑，不是整个地区的全部图斑。

### 2. 制作无人机航线

- 在地图上导入 KML/KMZ 图斑。
- 自动或手动规划航点。
- 调整航线和航点。
- 导出航线文件。

航线规划页面来自项目根目录的 `index.html`，部分地图资源需要联网加载。

### 3. 给照片加水印

- 扫描单个照片目录或包含其子文件夹。
- 读取照片 EXIF 中的拍摄时间、经纬度和尺寸。
- 在照片左下角添加标题、经度、纬度和拍摄时间。
- 自定义水印字号、文字颜色、描边颜色、透明度和边距。
- 支持添加多行自定义文字。
- 照片按拍摄时间排序后，从 `A1` 开始自动编号。
- 支持原图与处理效果切换预览。
- 自动处理同名文件，不覆盖原照片。
- 大量照片采用分页列表和预览缓存，降低界面卡顿。
- 提供“快速处理”和“兼容处理”两种模式。

### 4. 按图斑整理照片

- 选择一个或多个 KML 图斑文件。
- 单独选择需要整理的照片文件夹。
- 手动解析 KML，不依赖 GDAL/Fiona 的 KML 驱动。
- 根据照片 EXIF GPS 判断照片属于哪个图斑。
- 照片位于多个重叠图斑时，优先选择面积更小、更具体的图斑。
- 可设置“图斑外允许距离”，将图斑附近的照片分到最近图斑。
- 支持保留原照片（复制）或取走原照片（移动）。
- 提供双线程快速整理和单线程兼容整理。
- 文件重名时自动添加 `_1`、`_2` 等后缀。
- 只有实际匹配成功的图斑才建立照片文件夹。
- 未匹配照片不重复复制，只写入辅助 KML。

整理完成后会生成：

```text
选择的输出目录/
├── 图斑编号1/
│   ├── 照片1.jpg
│   └── 照片2.jpg
├── 图斑编号2/
│   └── ...
├── 无照片图斑.kml
├── 未匹配照片.kml
├── 图斑外距离匹配照片清单.txt
├── 图斑照片分类工作日志.txt
└── 整理结果.csv
```

匹配规则：

1. 只处理具有有效经纬度的照片。
2. GPS 点位于图斑内部或边界上时，直接匹配。
3. 未直接命中且允许距离大于 `0` 时，匹配阈值内最近的图斑。
4. 未找到符合条件的图斑时，照片写入 `未匹配照片.kml`。
5. 没有成功处理照片的图斑写入 `无照片图斑.kml`。

建议先使用 `0 米`查看结果。普通 GPS 存在少量偏差时，可尝试 `10～20 米`；距离设置过大会增加照片归错相邻图斑的风险。

### 5. 上传照片到平台

- 在软件内登录三资平台并自动识别登录状态。
- 下载页面与上传页面共享平台登录会话。
- 支持切换账号和退出登录。
- 选择本次照片对应的 KML。
- 扫描按图斑编号整理好的照片文件夹。
- 上传前核对：
  - 当前登录地区编号；
  - KML 中的图斑编号；
  - 照片文件夹名称中的图斑编号。
- 任一编号不一致时阻止整批上传，避免把照片传到错误地区。
- 可设置每个图斑最多上传的照片数。
- 可跳过平台已有的同名照片。
- 可从前、中、后位置均匀选择照片。
- 先检查、后上传；检查阶段不会向平台写入照片。
- 上传结果可保存为表格。

账号和密码只在平台网页中输入，本软件不保存账号密码。登录会话由软件内置浏览器配置保存，用户可以主动切换账号或退出登录。

### 6. 查看照片地图

- 加载照片并查看 GPS 点位。
- 加载 KML/KMZ 图层。
- 加载航线。
- 搜索图斑名称或编号。
- 显示、隐藏和清理地图图层。

照片地图页面来自项目根目录的 `gps_map.html`。

## 界面设计

软件左侧按照实际工作顺序排列六个页面：

1. 下载平台图斑
2. 制作无人机航线
3. 给照片加水印
4. 按图斑整理照片
5. 上传照片到平台
6. 查看照片地图

各页面提供简化后的中文说明、操作步骤、处理状态、进度提示和结果弹窗，尽量减少需要理解的开发术语。

## 环境要求

- Windows 10 或 Windows 11
- Python `3.12` 或 `3.13`
- [uv](https://docs.astral.sh/uv/)
- 能正常访问三资平台和地图资源的网络环境（仅联网功能需要）

项目使用独立虚拟环境，不应把依赖安装到全局 Python。

## 安装与运行

### 使用 uv

在项目目录中执行：

```powershell
uv sync
uv run sanzi-photo-tool
```

安装开发和测试依赖：

```powershell
uv sync --extra dev
```

运行测试：

```powershell
uv run pytest
```

### 使用项目启动脚本

环境准备完成后，也可以双击：

```text
启动软件.bat
```

## 核心依赖

| 依赖 | 用途 |
| --- | --- |
| PySide6 | Windows 桌面界面和内置网页 |
| Pillow | 图片读取、EXIF 解析和水印生成 |
| Shapely | KML 多边形、空间索引和照片点位匹配 |
| uv | 项目环境与依赖管理 |
| pytest | 自动化测试，仅开发环境使用 |

为了提升打包环境的稳定性，KML 使用 `xml.etree.ElementTree` 手动解析；坐标投影采用项目内的 Web Mercator 转换，不依赖 GDAL/Fiona 或后台线程中的 PROJ DLL。

## 数据安全与注意事项

- 水印和整理功能默认不会覆盖原照片。
- “取走原照片”会移动源文件，使用前会再次确认。
- 建议每次整理使用一个新的空输出目录，避免旧结果和新结果混合。
- 上传前必须先查看检查结果，确认登录地区、KML 和照片文件夹编号一致。
- `图斑外允许距离`不是越大越好，设置过大会提高错配风险。
- 软件无法为没有 GPS 的照片判断所属图斑。
- 平台接口、网页结构或登录方式发生变化时，下载和上传功能可能需要同步适配。
- 请只处理和上传自己有权限使用的数据。

## Windows 正式打包

正式版采用 `Nuitka Standalone + Inno Setup`：

- Python 业务代码由 Nuitka 转换并编译为本机程序，提高源码逆向门槛；
- Qt、Qt WebEngine、Pillow 和 Shapely 运行库随程序发布；
- Inno Setup 生成全中文标准安装程序；
- 同时生成一个解压即可运行的免安装版。

准备项目构建环境：

```powershell
uv sync --group build
```

电脑还需要安装 Inno Setup 6。准备完成后运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_tools\build_release.ps1
```

构建结果位于 `release` 目录：

```text
三资辅助软件_1.0_安装程序.exe
三资辅助软件_1.0_免安装版.zip
发布文件校验值_SHA256.txt
```

安装包、免安装版和编译中间目录不会提交到 Git。
